Areas de GIT

Working Directory

Es el área de trabajo donde vamos a crear, editar, eliminar los diferentes archivos que formen parte de nuestro trabajo. Muy rara vez estaremos utilizando la consola en esta instancia que básicamente estaría representada en la carpeta donde inicializamos el repositorio.

  • dir || ls Muestra el contenido del directorio actual.
  • mkdir <name Crea una carpeta.
  • cd <path Abre la carpeta o ruta seleccionada.
  • git init Inicializa un repositorio de GIT.
  • rm -rf .git < Elimina un repositorio de manera forzada.

Staging Area (INDEX)

Es el área de control de cambios donde se realizan las capturas de código (snaptshots) y básicamente donde indicamos a que archivos realizamos seguimiento. Una vez realizados los cambios solamente debemos confirmarlos para agregarlos al repositorio.

  • git status Muestra el estado del archivo en relación al INDEX
    • untracked No se está haciendo seguimiento. En caso de pérdida es irrecuperable.
    • staged: Se realizó la captura del código, pero no se confirmó.
    • commited: Se confirmó la captura y se almacenaron los cambios en el historial.
    • modified: El archivo difiere con respecto a la última confirmación.
  • git add <file Agrega los cambios del archivo al INDEX.
  • git diff Muestra las diferencias entre el INDEX y el Working Directory.
  • git restore --staged <file Elimina la última captura realizada.
  • git rm --cached <file Quita el archivo del INDEX, no hace más seguimiento.

Repository

Es el área donde se almacenan las confirmaciones de los cambios realizados en archivos tipo BLOB (Binary Large Object File), se utilizan como puntos de referencia a los que volvemos siempre y cuando indiquemos a GIT cual es el número HASH que queremos visualizar.

  • git commit Abre el editor para realizar la confirmación de cambios.
  • git log Muestra el historial de confirmación de manera descendente.
  • git blame <file Muestra el autor, el número de HASH, fecha y hora en que se modificó cada línea del archivo.
  • git checkout <ref Mueve el apuntador a la posición seleccionada.
  • git fetch <remote Captura el historial de cambios del repositorio remoto.
  • git push <remote> <branch Sube los cambios de la rama seleccionada al repositorio remoto.
