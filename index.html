Areas de GIT

Working Directory

Es el área de trabajo donde vamos a crear, editar, eliminar los diferentes archivos que formen parte de nuestro proyecto. Muy rara vez estaremos utilizando la consola en esta instancia que básicamente estaría representada por la carpeta donde inicializamos el repositorio.

Staging Area (INDEX)

Es el área de control de cambios donde se realizan las capturas de código (snapshots) y básicamente donde indicamos a que archivos vamos a estar realizando seguimiento. Una vez agregados los cambios solamente debemos confirmarlos para agregarlos al repositorio.

Repository

Es el área donde se almacenan las confirmaciones de los cambios realizados en archivos de tipo BLOB (Binary Large Object File), se utilizan como puntos de control a los que podemos volver, siempre y cuando indiquemos a GIT cual es el número de HASH que queremos visualizar.

  • dir || ls muestra el contenido del directorio actual
  • mkdir <name> crea una carpeta
  • cd <path> abre la carpeta o ruta seleccionada
  • git init inicializa un repositorio de git
  • rm -rf .git elimina el repositorio de manera forzada
